# 财经新闻情绪与市场波动日报系统｜复试项目讲稿（中文版 V1）

## 一、项目一句话介绍
我做了一个**财经新闻情绪与市场波动日报系统**，它会每天自动抓取公开财经新闻，经过清洗、情绪分析和市场数据联动后，生成日报、图表，并同步展示到 GitHub Pages。这个项目的重点不是做交易预测，而是把**金融信息处理、文本分析和市场观察**结合起来，做成一个可以持续运行、也可以在面试中清楚讲述的金融科技项目。

---

## 二、为什么做这个项目
我本科是金融科技专业，想在复试里更具体地展示自己把技术方法落到金融场景中的能力。

一开始我就在想，什么样的项目既能体现我的金融科技背景，又不至于做得太重、最后讲不清楚。后来我选了“财经新闻情绪与市场波动日报系统”，主要有三个原因：

1. **金融场景明确**：财经新闻会影响市场预期，这个题目和金融信息处理天然相关；
2. **技术链路完整**：从数据抓取、文本清洗、情绪打分，到市场联动、图表生成、自动化发布，是一条比较完整的工程链路；
3. **适合持续展示**：它不是一次性作业，而是能每天更新、持续积累历史样本，更适合放在 GitHub 上长期展示。

所以这个项目本质上是我把“金融科技能力”具体化、可视化的一次尝试。

---

## 三、项目做了什么
目前这个项目已经能稳定完成下面几个步骤：

1. **抓取财经新闻标题**
   - 从公开财经新闻源获取当天新闻；
   - 也支持对最近一周做历史回补，便于补齐展示层样本。

2. **清洗和去噪**
   - 过滤活动页、宣传稿、论坛页、路演页、重复标题等低信息密度内容；
   - 尤其对历史回补数据，单独加强了过滤规则，避免政策宣传或机构稿件把情绪结果抬得过高。

3. **做情绪分析**
   - 不是只靠单一模型判断，而是采用 **SnowNLP + 财经规则词典** 的混合方法；
   - 比如把“战争、冲突、制裁、油价飙升、通胀、加息”这类词更明确地压向负面；
   - 把“增长、改善、分红、上调评级”等偏积极信号归到正面。

4. **和市场数据联动**
   - 当前主要对接沪深300和上证指数日收益率；
   - 不仅看单日情绪，也加入了**3日滚动情绪**，因为新闻对市场往往存在滞后性和持续发酵效应。

5. **自动生成展示结果**
   - 每天自动生成日报、图表和 GitHub Pages 页面；
   - 首页可以直接看到最新情绪快照、主题分布、历史变化和情绪-市场联动图。

## 四、这个项目里我觉得最关键的难点
我觉得最难的点其实不是“把代码跑起来”，而是**把结果做得更可信、更像金融场景**。

因为一开始如果直接用通用情绪模型，很容易出现明显误判。比如：
- “伊朗战争导致油价上涨”可能会被误判成正面；
- “机构宣传稿、金融活动稿”会把整体情绪抬得过高；
- 如果只看当天新闻和当天市场，也会忽略新闻影响的滞后性。

所以我后面重点做了三轮修正：

1. **加强清洗规则**，压掉宣传稿、活动稿、路演页等噪声；
2. **强化财经语境下的情绪词典**，修正常见误判；
3. **加入滚动情绪视角**，让情绪与市场联动更符合实际。

这让我意识到，金融科技项目不只是“用了模型”，更重要的是要结合业务语境不断修正方法。

## 六、项目的局限性
我也很清楚它现在还有局限：

1. **它不是交易预测模型**，更适合作为信息面和风险偏好的观察面板；
2. **新闻源仍存在样本偏差**，尤其历史回补数据和日常抓取源口径还不完全一致；
3. **情绪分类仍然是轻量方案**，虽然比纯模型更稳，但还不是严格意义上的高精度金融文本分类器；
4. **历史样本还不够长**，所以当前更适合做展示和观察，不适合下强统计结论。

我觉得恰恰是因为我能看到这些局限，所以这个项目才更真实，而不是一个看起来很全但经不起追问的项目。
