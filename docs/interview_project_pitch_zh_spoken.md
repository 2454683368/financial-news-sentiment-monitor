# 财经新闻情绪与市场波动日报系统｜中文口语化面试稿 V1

## 1. 老师如果说：你介绍一下你的项目
老师您好，我做的这个项目叫**财经新闻情绪与市场波动日报系统**。简单来说，它每天会自动抓取公开财经新闻，先做标题清洗和去噪，再做情绪分析，然后和沪深300、上证指数这些市场数据做一个对照，最后自动生成日报、图表和 GitHub Pages 展示页面。

我做这个项目的初衷，其实是想把自己本科金融科技的背景，落到一个更具体、可展示、也能讲清楚的金融场景里。因为如果只是说我学过 Python、学过机器学习，会比较空；但如果我能拿出一个真实在跑、还能持续更新的项目，老师会更容易看到我把技术方法用到金融信息处理中的能力。

这个项目我最想强调的不是“预测涨跌”，而是**观察信息、理解情绪、分析市场风险偏好**。因为我觉得对于我现在这个阶段来说，做一个轻量但完整、而且逻辑上能讲明白的项目，比硬做一个很重但讲不清的量化系统更适合。

---

## 2. 老师如果说：你具体做了哪些工作
这个项目我主要做了四块工作。

第一块是**数据获取**。我会从公开财经新闻源抓取新闻标题，后面为了让历史图表更完整，我还加了历史回补逻辑，去补最近一周的数据。

第二块是**数据清洗**。因为原始新闻里有很多噪声，比如活动稿、宣传稿、论坛页、路演页，或者一些低信息密度的稿件。如果这些内容不处理，情绪结果会被带偏。所以我专门加了标题过滤、去重、域名排除这些规则。

第三块是**情绪分析**。我没有只用一个通用模型直接判断，而是用了 SnowNLP 和财经规则词典的混合方案。因为我后来发现，通用模型在财经语境里会有明显误判。比如“战争导致油价上涨”里出现“上涨”，它可能误判成正面，但其实从市场风险偏好角度看，应该更偏负面。所以我对像战争、冲突、制裁、通胀、加息、油价飙升这些词做了专门修正。

第四块是**展示和自动化**。现在这个项目已经能自动生成日报、历史图表和 GitHub Pages 页面，也就是它不只是本地跑通，而是能持续更新和展示。

---

## 3. 老师如果说：这个项目最大的难点是什么
我觉得最大的难点，不是把代码写出来，而是让结果**更可信、更符合金融语境**。

因为一开始项目虽然能跑，但结果不够像一个金融项目。最典型的问题是，某些明显偏负面的新闻，比如地缘冲突、油价飙升、加息预期，模型会因为“上涨、突破”这类词，把它误判成正面。还有一类问题是，历史回补的数据里会混进很多宣传稿、政策活动稿，这些内容天然偏正，也会把整体情绪抬高。

所以我后面做的重点，其实不是“多加功能”，而是不断修正结果。比如：
- 加强清洗规则，压低噪声；
- 加强财经规则词典，修正误判；
- 加入 3 日滚动情绪，考虑新闻影响的滞后性；
- 对历史回补源再做单独过滤。

我觉得这部分反而最能体现我对项目的理解，不是停留在“把模型跑通”，而是会根据金融场景不断校正。

## 4. 老师如果说：你觉得这个项目体现了你什么能力
我觉得主要体现了三点。

第一是**金融场景理解能力**。我不是把文本情绪分析当成一个纯 NLP 任务，而是一直在围绕“新闻—预期—市场表现”这条逻辑去做项目。

第二是**技术落地能力**。我把新闻抓取、清洗、情绪分析、市场联动、图表生成、自动发布串成了一个完整流程，而不是只停留在单个脚本或一个 demo 上。

第三是**迭代优化能力**。这个项目不是一开始就很完善，而是我在实际跑数据的过程中不断发现问题、修正问题。这让我更理解：金融科技项目很多时候不是模型本身有多复杂，而是你能不能把方法放到真实场景里持续优化。

---

## 5. 老师如果说：这个项目有什么局限性
我会比较坦诚地说，这个项目现在有几个明显局限。

第一，它**不是交易预测模型**，更适合作为新闻情绪和市场风险偏好的观察面板。

第二，新闻源本身会有样本偏差，尤其历史回补和日常抓取源目前还不是完全统一口径，所以我更倾向把它当成一个可解释的观察系统，而不是严格的量化结论。

第三，情绪分析现在是一个轻量混合方案，优势是可解释、易迭代，但精度肯定不如专门做金融文本的大模型或人工标注体系。

我觉得这些局限并不可怕，关键是我知道问题在哪，也知道后面该怎么继续优化。

---

## 6. 老师如果说：为什么不直接做量化交易或者更复杂的大模型项目
我会这样回答。

因为我现在更希望做的是一个**和自己背景匹配、完成度高、而且能讲清楚的项目**。如果直接做一个很重的量化交易系统或者大模型项目，短时间内可能很难做到真正有完成度，而且一旦老师追问数据、策略、回测、因子、稳健性，我反而容易讲空。

相比之下，这个项目的优点是：场景明确、链路完整、可持续更新，而且和金融科技背景贴合。我觉得对于复试来说，一个真实、完整、可解释的项目，比一个看起来很大但讲不实的项目更有说服力。

## 7. 1分钟精简版
老师您好，我做了一个财经新闻情绪与市场波动日报系统。它每天自动抓取公开财经新闻，经过清洗、情绪分析和市场数据联动后，生成日报、图表和 GitHub Pages 展示页面。

我做这个项目的初衷，是想把金融科技背景更具体地落到一个真实金融场景里。项目核心不是预测涨跌，而是观察新闻如何影响市场预期和风险偏好。

在做的过程中，我重点解决了两个问题：一是新闻源里有很多宣传稿和噪声内容，需要先清洗；二是通用情绪模型在财经语境里会误判，所以我用了 SnowNLP 加财经规则词典的混合方案，并加入了 3 日滚动情绪来体现新闻影响的滞后性。这个项目让我更深地体会到，金融科技不是简单套模型，而是要结合具体场景不断修正方法。
